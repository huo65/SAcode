# 待完善功能

## 1. 分析范围

本清单依据以下材料交叉整理：

- `第3组-软体实践材料1.pptx`：需求分析、非功能需求、体系结构选择。
- `第3组-软体实践材料2.pptx`：用例分析、质量属性场景、体系结构评估与风险决策。
- `code/front/src`：Vue 前端页面、路由、状态管理、API 封装。
- `code/backend/DB-market/src/main/java`：Spring Boot 后端 Controller、Service、Mapper、Pojo。

当前项目已经具备基础外卖商城雏形：登录/注册、商品浏览与搜索、商家商品管理、商品审核、购物车、下单、模拟支付、订单状态流转、个人信息与地址维护、分类管理等。但 PPT 中描述的是较完整的外卖订单系统，仍有不少顾客端、商家端、骑手端、管理端和非功能需求尚未闭环。

## 2. 当前已基本覆盖的能力

### 2.1 用户与角色

- 已有用户名/密码登录接口：`/user/login`。
- 已有用户注册接口：`/user/register`。
- 前端状态支持 `customer`、`merchant`、`driver`、`admin`、`guest`。
- 注册页面目前只开放顾客 `cus` 与商家 `mer` 两类。
- 已有用户信息查询与修改：`/user/getinfo`、`/user/update`。
- 已有地址新增、查询、删除：`/user/addAdress`、`/user/address`、`/user/deleteAddress`。

### 2.2 商品与菜单

- 已有商品搜索：分类、名称、价格区间、价格排序、商品状态筛选。
- 顾客只能检索审核通过商品，商家可看自己商品或已通过商品，管理员可看全部商品。
- 商家可新增、编辑、删除商品。
- 管理员可审核商品：通过或拒绝。
- 已有分类查询、新增、删除。

### 2.3 购物车、下单与支付

- 顾客可添加商品到购物车、修改购物车数量、批量下单。
- 顾客可直接购买单个商品。
- 下单时可选择收货地址。
- 已有订单创建、订单支付、订单列表查询、订单状态更新。
- 前端提供微信、支付宝、PayPal 三种支付入口样式，但主要是扫码模拟支付。
- 后端存在支付宝沙箱相关接口：`/alipay/pay`、`/alipay/notify`、`/alipay/refund`、`/alipay/check`。

### 2.4 订单状态

已建模的状态包括：

- `-3`：已退货/退款
- `-2`：退货中
- `-1`：下单未支付
- `0`：已支付/待发货
- `1`：配送中
- `2`：已收货
- `3`：待骑手接单

前端订单页已有付款、商家发货、骑手接单、顾客确认收货、顾客申请退货、商家确认退货等按钮。

## 4. 顾客端待完善

PPT 需求：

- 支持手机号、社交媒体账号或邮箱注册和登录。
- 忘记密码时可重置密码。
- 注册信息错误或不符合要求时提供错误提示。

源码现状：

- 前端登录只提供 `name` + `password`。
- 注册字段包含账号类型、用户名、手机号、密码、确认密码。
- 后端按用户名和密码直接查询登录。

待完善：

- 增加手机号/邮箱登录方式。
- 增加短信或邮箱验证码注册与找回密码流程。

待完善：

### 4.3 订单创建

## 9. 架构风险与技术债

### 9.1 单一数据库风险

PPT 已指出：单一数据库故障会影响系统稳定性和可用性。

待完善：

- 增加数据库备份。
- 后续支持主从复制和读写分离。
- 为订单、支付、退款等核心链路增加幂等和补偿机制。

### 9.2 图片存储风险

PPT 提到 Base64 会带来约 33% 存储膨胀。

源码现状：

- 当前不是完整 Base64 落库，而是保存 `/img/文件名`。
- 但前端没有真正上传文件，部署后图片资源不可控。

待完善：

- 使用文件系统或对象存储保存真实图片。
- 数据库只保存 URL 和元数据。
- 对小图可考虑压缩或缩略图策略。
- 增加图片安全校验，避免任意文件名引用。

### 9.3 订单状态机不严格

源码现状：

- `/order/update` 允许传入任意合法范围内的状态。
- 未看到基于当前状态、角色、业务规则的状态迁移校验。

待完善：

- 明确定义状态迁移表。
- 校验只有允许角色能触发允许迁移。
- 支付、退款、接单等关键操作应由专门服务驱动状态变化。

### 9.4 越权风险

源码现状：

- 管理员、商家、顾客、骑手的权限主要在前端控制。
- 后端根据传入 id 查询和更新，缺少 token 身份绑定。

待完善：

- 后端从 token 中解析当前用户，不信任前端传入的用户 id。
- 商品编辑/删除必须校验商家归属或管理员权限。
- 订单查询/更新必须校验订单参与方或管理员权限。
- 地址增删改查必须校验归属。
