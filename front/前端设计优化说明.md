# 前端设计优化说明

## 1. 现状问题清单

- 全局视觉基底过于单一：页面长期使用高饱和纯黄底色，缺少层次变化、品牌识别和场景氛围。
- 字体层级不清晰：标题、正文、说明文字和数据指标缺少统一字号与字重规则，页面信息主次不够明确。
- 页面骨架割裂：登录页、头部、顾客页、骑手页、商家页和管理页各自为战，缺少统一的导航、容器和留白体系。
- 组件风格偏后台默认态：表单、表格、Tab、弹窗等大多沿用默认样式，卡片感、质感和状态反馈不足。
- 排版偏“平铺式”：大量内容直接堆叠在白底容器里，空间利用率低，缺少分栏、卡片组、概览区和信息摘要区。
- 交互反馈偏生硬：悬停、切换、打开弹窗、操作按钮反馈较弱，用户难以感知当前操作是否完成。
- 响应式薄弱：部分页面仍以固定最小宽度为主，移动端和窄屏下容易出现拥挤、换行混乱和信息截断。
- 顾客与骑手端不够“产品化”：顾客页更像管理后台 Tab，骑手页虽然已有数据面板，但整体仍缺少驾驶舱式视觉组织。
- 核心信息密度失衡：订单详情信息量大，但缺少视觉分段与重点突出，阅读成本偏高。
- 组件一致性不足：相同语义的区域在不同页面使用了不同圆角、背景、边框和排版方式。

## 2. 统一设计规范

### 2.1 品牌色彩体系

- 主品牌色：暖橙棕系，传达外卖、餐饮和热食场景。
- 强调色：高亮琥珀金，用于关键指标、微标签和局部聚焦。
- 中性色：暖白、米杏、浅咖，用于页面背景、卡片分层和弱对比边界。
- 语义色：
  - 成功：柔和绿，用于营业中、完成、在线状态
  - 警告：暖黄橙，用于待处理、忙碌、提醒
  - 危险：柔和红，用于超时、禁用、退款和风险动作

### 2.2 字体层级

- 展示标题：使用衬线风格字体栈，突出品牌感和页面记忆点。
- 正文与表单：使用清晰的人文无衬线字体栈，确保长文本和后台信息可读。
- 层级建议：
  - 页面主标题：`30px - 46px`
  - 模块标题：`22px - 30px`
  - 正文：`14px - 16px`
  - 辅助说明：`12px - 13px`
  - 数据指标：`24px - 30px`

### 2.3 组件样式标准

- 容器：统一使用大圆角玻璃卡片，带轻边框和柔和阴影。
- 按钮：统一圆角胶囊化，主按钮使用暖橙渐变，辅按钮保持浅底高可读。
- 输入框：统一圆角、浅底、聚焦高亮环，减少默认后台感。
- Tab：改为胶囊式导航，强化当前选中态而非默认下划线。
- 表格：弱化硬边框，增强表头区分与行 hover 反馈。
- 弹窗与抽屉：统一暖色玻璃层叠风格，保证与主界面一致。
- 标签：统一圆角胶囊态，用颜色快速表达状态，不依赖冗长文字。

### 2.4 动效原则

- 全站统一过渡时长以 `180ms - 260ms` 为主。
- 优先加入 hover、focus、卡片抬升、阴影增强等低成本高收益动效。
- 避免大面积复杂动画，优先保证列表滚动、弹窗打开和表单输入流畅。
- 尊重低性能和减少动画场景，避免影响课堂展示稳定性。

## 3. 本次优化落地范围

- 全局设计令牌与 Element Plus 皮肤重构
- 登录页重构为品牌化双栏入口
- 顶部头部导航重构
- 顾客、商家、骑手、管理员首页骨架统一
- 商品、订单、个人信息、分类、用户管理模块卡片化与分区优化
- 按钮、表单、表格、标签、弹窗和 Tab 的动效与状态反馈升级

## 4. 后续建议

- 继续补齐 `after-sale`、`operation`、`restaurant-dialog` 等深层弹窗和工作台的视觉统一。
- 根据课堂展示流程，再补一套“演示模式”引导提示文案和空状态插画占位。
- 若后续允许引入品牌字体资源，可进一步提升登录页和首页的识别度。
