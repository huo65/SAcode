# 功能完善进度

本文档对应 `待完善功能.md`，用于记录当前开发进度、已落地改动、验证状态和后续待办。

文档协同关系如下：

- `AGENTS.md`：定义当前项目以课堂展示为导向的总体目标、开发原则、联调经验与取舍边界。
- `功能完善进度.md`：记录模块进度、路线图、验证状态和阶段验收记录。
- `Phase1联调验收文档.md`：记录 Phase 1 的初始化步骤、四角色最小联调主流程、操作清单和联调记录模板。

后续更新时，应按职责范围维护文档：

- 目标、原则、取舍边界变化时，更新 `AGENTS.md`。
- 模块状态、路线图、验证状态、待办变化时，更新 `功能完善进度.md`。
- 仅当 Phase 1 的初始化步骤、联调流程、操作清单、验收记录或当前结论变化时，才更新 `Phase1联调验收文档.md`。
- 若一次改动同时影响多个范围，再同步更新对应文档，避免出现“原则、状态、验收清单”彼此不一致。

## 2. 已完成改动

### 2.1 认证授权

- 启用 `LoginInterceptor`，除登录、注册、支付宝回调和匿名商品/分类浏览外，其余接口需要 `Token`。
- JWT claims 增加 `id`、`name`、`type`，后端通过 `CurrentUserHolder` 获取当前用户。
- 前端登录成功后保存 token 和用户信息，刷新后可恢复基础登录态。
- 请求返回 `code=-1` 时，前端自动清理登录态并跳回登录页。
- 增加全局 CORS，允许登录后携带自定义 `Token` 请求头访问各模块接口。
- 新注册密码使用 SHA-256 哈希；旧明文密码登录成功后自动迁移为哈希。

### 2.2 订单、支付、库存

- `/user/order` 下单时由后端校验商品存在、已审核、库存充足、数量有效，并由后端计算订单金额。
- `/cart/makeOrder` 继续支持按商家拆单，同一商家多商品共用订单号。
- `/user/pay` 改为只接收 `orderIdList`，不再接受前端传入模拟价格。
- 支付成功后后端扣用户余额、扣商品库存，并将订单状态更新为已支付待商家处理。
- `/order/update` 增加状态机校验，不再允许前端任意传状态直接覆盖。
- 已支持状态流转：待支付 `-1`、已支付 `0`、备餐中 `4`、待骑手接单 `3`、配送中 `1`、已完成 `2`、退款中 `-2`、已退款 `-3`。
- 顾客申请退款、商家或管理员确认退款后，后端回滚订单金额到顾客余额。
- 商家可直接拒绝已支付订单，系统自动退款给顾客并恢复库存。
- 商家可取消备餐中的订单，系统自动退款给顾客并恢复库存。
- **【Phase1完成】** 支付时使用 `SELECT ... FOR UPDATE` 悲观锁，防止并发库存超卖。
- **【Phase1完成】** 退款确认时自动恢复商品库存。

### 2.3 角色与权限

- 顾客只能操作自己的购物车、地址、订单。
- 商家只能新增商品、编辑/删除自己的商品，并处理自己的订单。
- 骑手只能查看待接单或自己负责的配送订单，并从待接单状态接单。
- 管理员可查看全部订单、审核商品、查看用户列表、启用/禁用用户。
- 匿名商品搜索不再信任前端传入的用户 ID，避免伪造角色查看未审核商品。

### 2.4 前端页面

- 修复登录 token 字段不一致导致登录后立刻掉线的问题。
- 注册页增加骑手类型。
- 骑手端移除商品浏览，保留订单和个人信息。
- 支付组件改为新接口请求体 `{ orderIdList }`。
- 商品新增表单修复传入 `ref` 对象而不是表单值的问题。
- 订单页适配 `imgList` 和商品列表数量显示。
- 管理端新增用户管理页，支持角色筛选和启用/禁用账号。
- 顾客端单商品下单和购物车批量下单已支持订单备注、期望送达时间填写，并在订单列表展示。
- 新增 `/restaurant/list`、`/restaurant/info` 聚合接口，顾客首页改为餐厅列表入口，支持餐厅详情查看菜单并继续复用现有菜品下单弹窗。
- 新增订单评价表与 `/review/add`、`/review/reply` 接口，顾客可对已完成订单评价一次，商家可回复评价，餐厅详情页可展示评分均值、评价列表与回复内容。
- 商家订单页支持接单、备餐完成后推送骑手、拒单、取消备餐中订单；拒单/取消时需填写原因并回写到订单退款原因。

- 骑手配送中的订单支持主动拒单，拒单后订单回到待接单池并清空 `driver_id`，其他骑手可继续接单。
- 骑手首页新增配送概览卡片，展示待抢单、配送中、今日完成和累计配送收入；收入按课堂展示版规则 `max(4, 订单金额 × 10%)` 估算。
- 骑手首页新增在线接单/休息中切换，休息状态下隐藏接单按钮，并在前端阻止骑手接新单。
