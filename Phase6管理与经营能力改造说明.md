# Phase 6 管理与经营能力改造说明

## 1. 改造目标

按照课堂展示版验收标准，补齐 `Phase 6: 管理与经营能力`，让项目具备以下可连续演示的能力：

- 管理端可统一查看平台经营概览、订单趋势、角色规模与近期样本订单。
- 管理端可通过轻量 RBAC 页面调整管理员/商家的菜单与关键动作权限，并在前端菜单与后台关键接口上即时生效。
- 商家端可查看门店经营分析看板，包括 GMV、有效订单、平均客单价、售后结构、热销商品与近 7 日趋势。
- 平台与商家关键操作都能留下审计日志，便于课堂展示“治理留痕”。

本阶段继续遵循课堂展示导向，不追求企业级复杂 RBAC 或 BI 系统，而是优先交付可讲解、可核对、可回放的最小闭环。

## 2. 方案取舍

### 2.1 权限系统

- 不引入完整的角色继承、菜单树和接口资源中心。
- 采用单表 `operation_permission` 存储“角色-权限键-菜单/动作-开关”。
- 权限键分为两类：
  - `menu`：控制管理端/商家端标签页是否展示。
  - `action`：控制关键管理动作和 `ops` 相关后台接口是否允许执行。
- 课堂展示重点放在“调整权限 -> 菜单变化 -> 关键动作被拦截”这一条最容易讲清楚的链路。

### 2.2 经营报表

- 不接入第三方图表库，避免额外复杂度与兼容问题。
- 平台与商家的统计数据直接复用当前订单、商品、用户、售后数据，在服务层做聚合计算。
- 趋势图采用前端 CSS 可视化，保证 `npm run build` 稳定通过且页面呈现直观。

### 2.3 审计日志

- 不做完整操作上下文还原和分页检索系统。
- 采用单表 `operation_audit_log` 记录关键动作：权限调整、商品审核、售后处理、门店资料更新、账号启停。
- 管理端查看全量日志，商家端查看本店/本人经营相关日志即可满足课堂演示。

## 3. 后端改造

### 3.1 新增数据表

- `operation_permission`
  - 角色编码
  - 权限键
  - 权限名称
  - 权限类型（menu/action）
  - 作用域（admin/merchant）
  - 启用状态
  - 更新时间
- `operation_audit_log`
  - 操作人信息
  - 动作类型
  - 目标对象
  - 动作描述
  - 处理结果
  - 创建时间

同时更新 `schema.sql`，并在运行时通过 mapper 的 `createTableIfMissing()` 保证旧库也能平滑补表。

### 3.2 新增模块

- `OperationsController`
  - `/ops/me`
  - `/ops/permission/list`
  - `/ops/permission/update`
  - `/ops/admin/dashboard`
  - `/ops/merchant/dashboard`
  - `/ops/merchant/export`
  - `/ops/audit/list`
- `OperationsService` / `OperationsServiceImpl`
  - 权限种子初始化
  - 权限快照读取
  - 平台经营数据聚合
  - 商家经营数据聚合
  - CSV 导出内容组装
  - 审计日志写入与查询

### 3.3 关键联动

为确保“权限配置立即生效”不仅停留在前端展示层，本轮还在关键管理动作中接入权限判断与审计记录：

- `UserController`
  - 用户列表查看
  - 账号禁用/启用
- `ProductController`
  - 商品审核
- `RestaurantController`
  - 商家门店资料查看/更新
- `AfterSaleController`
  - 商家/管理员处理售后工单

## 4. 前端改造

### 4.1 管理端

- 在管理端首页新增 `Ops` 标签。
- 重新组织首页为“治理中心”视觉结构，补充启用权限统计。
- 新增平台经营看板：
  - 平台 GMV
  - 有效订单
  - 角色规模
  - 订单状态分布
  - 最近订单样本
- 新增权限矩阵页面：
  - 管理员/商家两组配置
  - 菜单/动作开关
  - 更新后通过事件广播刷新当前菜单
- 新增审计日志列表。

### 4.2 商家端

- 在商家首页新增 `Ops` 标签。
- 新增经营分析页面：
  - 门店 GMV
  - 有效订单
  - 平均客单价
  - 售后结构
  - 热销商品排行
  - 近 7 日趋势
  - 最近操作日志
- 新增经营数据 CSV 基础导出。

### 4.3 权限驱动菜单

管理端与商家端首页不再写死所有标签页，而是：

1. 登录后拉取 `/ops/me`
2. 根据 `menuMap` 决定标签是否展示
3. 如果当前标签被关闭，自动切回第一个可用标签
4. 当管理员修改权限时，通过浏览器事件通知当前页面重新拉取权限快照

## 5. 课堂展示建议流程

推荐按以下顺序演示：

1. 管理员进入 `Ops` 页面，展示平台 GMV、订单趋势、角色规模和最近订单样本。
2. 在权限矩阵中关闭商家的 `merchant.menu.ops` 或 `merchant.action.report.export`。
3. 切换到商家端，展示对应标签页消失或导出动作被拦截。
4. 再恢复权限，展示商家端经营分析台与导出能力恢复可用。
5. 回到管理员端查看审计日志，说明权限调整与关键动作已留痕。

## 6. 后续可扩展项

- 接口级资源树、菜单树和角色继承。
- 更细维度的退款率/投诉率专题报表。
- 审计日志分页、导出与高级筛选。
- 多门店商家聚合统计与经营对比。
- 更完整的 BI 图表和报表中心。
